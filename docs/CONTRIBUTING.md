# 貢獻指南

感謝您對 BearCore-V 的興趣！歡迎貢獻。

## 開發流程

1. **報告問題**
   - 使用 GitHub Issues 報告 bug 或提出新功能建議
   - 提供詳細的重現步驟和環境信息

2. **提交代碼**
   - Fork 本倉庫
   - 創建功能分支：`git checkout -b feature/your-feature-name`
   - 遵循現有代碼風格
   - 添加適當的測試
   - 提交更改：`git commit -s -m "描述你的更改"`

3. **代碼風格**
   - Verilog: 使用 4空格縮進，模組端口每行一個
   - C語言: 遵循 K&R 風格
   - Python: 遵循 PEP 8
   - 註釋: 使用英文，關鍵演算法需詳細說明

4. **測試要求**
   - 所有新功能必須通過現有測試
   - 新增功能需提供對應測試程式
   - 性能修改後 CPI 不應顯著下降
